## Contributing guide
This document serves as a checklist before contributing to this repository. It includes links to additional information if topics are unclear to you.

This guide mainly focuses on the proper use of Git.

### 1. Issue

You are welcome to open an issue if you encounter any problems or have suggestions for this project. If you believe there is an issue with the sensor itself, please contact application support.

### 2. Pull Requests

ams is happy to receive contributions from the community.

#### How to proceed

* We recommend to engage first a communication through an issue, in order to present your proposal and just to confirm that it corresponds to the correct domain or scope.
* Then fork the project to your GitHub account to further develop your contribution. Please use the latest commit version.
* Please submit one Pull Request for one new feature or proposal. This will facilitate the analysis and the final merge if accepted.
